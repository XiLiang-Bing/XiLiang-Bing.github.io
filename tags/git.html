git 出现：warning: LF will be replaced by CRLF 的解决办法

2017-11-10

介绍

Windows 环境下使用 git add 的时候，有时会遇到 warning: “warning:LF will be replacee by CRLF” 提示。这是由于在 Windows 中的换行符为 CRLF，而在 Linux 中的换行符为 LF。在 git 创建的项目中换行符为 LF，而执行 git add 时，系统会提示 LF 将被转换为 CRLF。

这种问题的解决的办法很简单，在终端执行如下命令禁止 git 自动转换即可。

git config --global core.autocrlf false

Git https、http 方式设置记住用户名及密码

2017-11-8

Git 使用 https、http 方式时，每次都要手动输入密码，按照如下方法设置后，只需输入一次密码，以后再不用承受手动输入密码的困扰。

设置记住密码（默认15分钟）：

git config --global credential.helper cache

自定义密码失效时间（一小时之后失效）：

git config credential.helper 'cache --timeout=3600'

长期存储密码（推荐一）：

git config --global credential.helper store`

小提示：增加远程地址时加上用户名和密码也是可以的（推荐二）

git remote add origin http://username:password@git.oschina.net/name/project.git

补充：使用客户端也可以存储密码。

Git 常用命令

2017-11-7

  • 初始化一个版本库

    git init

  • 克隆远程版本库

    git clone <url>

  • 添加远程版本库

    git remote add origin <url>

  • 查看远程仓库

    git remote -v

  • 提交修改到暂存区，有三种模式可以选择（三选一）

    // 提交所有变化
git add -A
// 提交被修改(modified)和被删除(deleted)文件，不包括新文件(new)
git add -u
// 提交新文件(new)和被修改(modified)文件，不包括被删除(deleted)文件
git add .

  • 提交修改

    git commit -m "提交说明"

  • 推送到远程仓库

    git push origin master

  • 查看文件状态

    git status

  • 跟踪指定文件

    git add readme.txt

  • 从当前跟踪列表移除文件，并删除本地文件

    git rm readme.txt

  • 仅从暂存区删除，仍保留本地文件

    git rm -cached readme.txt

  • 重命名文件

    git mv reademe.txt readme.html

  • 查看提交历史

    git log

  • 修改最后一次提交的提交注释

    git commit --amend

  • 创建一个分支

    git branch <name>

  • 切换工作目录

    git chekcout <name>

  • 删除分支

    git branch -d <name>

  • 查看所有分支（有*号的为当前分支）

    git branch

  • 查看远程仓库的信息

    git remote show origin

  • 强制掩盖远程分支

     git push origin <name> --force
